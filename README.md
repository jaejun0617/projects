# Frontend Projects Monorepo

프론트엔드 개발자로 성장하기 위한 **개인 학습 · 실전 프로젝트 모노레포**입니다.  
단순 결과물 나열이 아니라, **기본기 → 구조 → 설계 → 확장성**을 기준으로 프로젝트를 관리합니다.

---

## 🎯 Repository Purpose

이 저장소의 목적은 다음과 같습니다.

- HTML / CSS / JavaScript / React 학습 결과를 **흐름 있게 기록**
- “동작하는 코드”보다 **이해 가능한 구조와 기준**을 우선
- 퍼블리셔 → 프론트엔드 개발자로 이어지는 **성장 과정 증명**

> 빠르게 만드는 것보다  
> **다시 봐도 설명할 수 있는 코드**를 목표로 합니다.

---

## 🧠 How I Think

### 1️⃣ 구조가 먼저다
- 마크업은 **시맨틱과 정보 구조**를 기준으로 작성
- 스타일은 “보이게”가 아니라 **역할 단위로 분리**
- JavaScript는 **UI를 보조하는 수단**일 뿐, 구조를 지배하지 않음

### 2️⃣ 재사용을 전제로 작성한다
- 컴포넌트 단위 사고 (Button / Card / Modal 등)
- 클래스 네이밍은 **확장 가능한 규칙** 유지
- CSS 역시 “한 페이지용 코드”가 아닌 **시스템 관점**으로 설계

### 3️⃣ 실무 기준을 학습 기준으로 삼는다
- 반응형: **모바일 우선 사고**
- 접근성: 키보드, 포커스, 시맨틱 기본 반영
- 성능: 불필요한 스타일 및 JavaScript 지양
