# JavaScript 핵심 이론 보강 — 변수(Variables) & 자료형(Types)

> 기준 문서: MDN JavaScript Guide — Grammar and types  
> 목표: “암기”가 아니라 **런타임에서 실제로 어떻게 동작하는지**를 기준으로 변수/자료형을 정리한다.

---

## 0) 한 줄 결론
- **변수는 “이름 → 값(또는 참조)”을 연결**하는 장치다.
- **자료형은 “값의 성격(연산/비교/저장 방식)”을 결정**한다.

---

## 1) 변수 선언 3종: `let` / `const` / `var`

### 1-1. 추천 규칙 (실무 기준)
- 기본은 **`const`**
- 값이 바뀌어야만 하면 **`let`**
- **`var`는 특별한 이유 없으면 금지** (레거시 코드 제외)

### 1-2. 왜 `var`를 피하나?
- 블록 스코프 미지원
- 선언 전 접근 시 에러가 아닌 `undefined`
- 디버깅 난이도 급상승

```js
if (true) {
  var x = 5;
}
console.log(x); // 5
```

```js
if (true) {
  let y = 5;
}
console.log(y); // ReferenceError
```

---

## 2) `undefined` vs 선언되지 않음

### 2-1. `undefined`
- 선언은 되었지만 값이 없음

```js
let a;
console.log(a); // undefined
```

### 2-2. 선언되지 않음
```js
console.log(b); // ReferenceError
```

## 3) 스코프(Scope)
- 전역 스코프
- 함수 스코프
- 블록 스코프 (`let`, `const`)

**실무 규칙**
- 전역 변수 최소화
- 데이터는 인자와 리턴으로 흐르게 설계

---

## 4) 호이스팅(Hoisting)

```js
console.log(x); // undefined
var x = 1;
```

```js
console.log(y); // ReferenceError
let y = 1;
```

> 결론: 호이스팅을 이용하지 말고 선언을 코드 상단에 명시

---

## 5) 자료형(Type)

### 5-1. 원시 타입 (Primitive)
- string
- number
- bigint
- boolean
- undefined
- symbol
- null

### 5-2. 객체 타입 (Object)
- object
- array
- function

```js
const a = { x: 1 };
const b = { x: 1 };
a === b; // false
```

---

## 6) `null` vs `undefined`

| 구분 | 의미 |
|---|---|
| undefined | 아직 값이 정해지지 않음 |
| null | 의도적으로 비워둠 |

```js
let u;
u + 2; // NaN

const n = null;
n * 2; // 0
```

---

## 7) typeof & 판별 팁

```js
typeof null; // "object" (주의)
Array.isArray([]); // true
```

---

## 8) `const`의 오해

```js
const obj = { x: 1 };
obj.x = 2; // 가능
obj = {};  // 에러
```
