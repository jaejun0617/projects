## 1) 연산자 (Operators)

### 1-1. 왜 연산자가 중요한가
연산자는 **값을 계산하는 도구가 아니라 흐름을 제어하는 도구**다.  
조건문, 상태 분기, UI 렌더링의 기준이 된다.

---

### 1-2. 산술 연산자
```js
+  -  *  /  %  **
```

```js
10 % 3;   // 1
2 ** 3;   // 8
```

⚠️ 실무 주의
```js
'10' + 2; // '102'
'10' - 2; // 8
```

→ **문자열 + 숫자 = 문자열 결합**

---

### 1-3. 비교 연산자 (중요)

```js
==   !=   // 타입 변환 발생 (비권장)
===  !==  // 타입까지 비교 (권장)
```

```js
0 == false;   // true ❌
0 === false;  // false ✅
```

✅ **실무 규칙: === / !== 만 사용**

---

### 1-4. 논리 연산자

```js
&&  ||  !
```

```js
const isLogin = true;
const isAdmin = false;

isLogin && isAdmin; // false
isLogin || isAdmin; // true
```

#### Short-circuit 평가 (매우 중요)

```js
const user = null;
user && user.name; // null (에러 방지)
```

```js
const title = input || '기본값';
```

---

### 1-5. Nullish Coalescing (??)

```js
const count = 0;

count || 10; // 10 ❌
count ?? 10; // 0 ✅
```

→ **0, '' 같은 유효값을 보존할 때 사용**

---

## 2) 조건문 (Conditionals)

### 2-1. if / else

```js
if (score >= 90) {
  grade = 'A';
} else if (score >= 80) {
  grade = 'B';
} else {
  grade = 'C';
}
```

📌 조건은 **명확하고 짧게**
- 중첩 if 지양
- 논리 연산자로 정리

---

### 2-2. 삼항 연산자 (조건 ? A : B)

```js
const label = isLogin ? '로그아웃' : '로그인';
```

⚠️ 실무 기준
- 한 줄 UI 상태 표현: 👍
- 중첩 삼항: ❌ (가독성 붕괴)

---

### 2-3. switch 문

```js
switch (type) {
  case 'coffee':
    price = 5000;
    break;
  case 'tea':
    price = 4500;
    break;
  default:
    price = 0;
}
```

📌 요즘은 **객체 맵 패턴** 선호

```js
const priceMap = {
  coffee: 5000,
  tea: 4500,
};

priceMap[type] ?? 0;
```

## 3) 함수 (Functions)

### 3-1. 함수의 역할
- 로직 묶기
- 중복 제거
- 책임 분리

> 함수는 “작게, 명확하게”

---

### 3-2. 함수 선언 vs 표현식

```js
// 선언식 (호이스팅 O)
function sum(a, b) {
  return a + b;
}

// 표현식 (호이스팅 X)
const sum2 = function (a, b) {
  return a + b;
};
```

실무에서는 **표현식 + const** 선호

---

### 3-3. 화살표 함수 (Arrow Function)

```js
const sum = (a, b) => a + b;
```

```js
const isAdult = age => age >= 20;
```

⚠️ this 없음
```js
// 객체 메서드에는 부적합한 경우 있음
```

---

### 3-4. 매개변수 & 기본값

```js
function greet(name = '손님') {
  return `안녕하세요 ${name}`;
}
```

---

### 3-5. return의 의미

```js
function check(age) {
  if (age < 0) return;
  return age >= 20;
}
```

- return 없으면 undefined
- 조건 분기 조기 종료에 적극 활용

---

### 3-6. 순수 함수 (Pure Function)

```js
function add(a, b) {
  return a + b;
}
```

특징
- 외부 상태 변경 ❌
- 같은 입력 → 같은 출력

→ 테스트/유지보수 최강
