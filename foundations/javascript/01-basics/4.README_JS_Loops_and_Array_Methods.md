# JavaScript 핵심 이론 — 객체(Object) & 구조 분해(Destructuring)

이 문서는 **실무 관점**에서 JavaScript 객체와 구조 분해를 이해하도록 설계된 학습용 README입니다.
문법 암기가 아니라 **데이터 구조를 읽고, 꺼내고, 재조합하는 사고**에 초점을 둡니다.

---

## 1. 객체(Object)란 무엇인가

객체는 **의미 있는 데이터의 묶음**이다.

```js
const user = {
  id: 1,
  name: "Jaehun",
  role: "frontend",
  active: true,
};
```

- 배열: **순서 중심**
- 객체: **의미(키) 중심**

👉 실무 데이터는 대부분 객체 기반(JSON)

---

## 2. 객체 기본 접근 방식

### 점 표기법 (dot notation)

```js
user.name; // "Jaehun"
```

### 대괄호 표기법 (bracket notation)

```js
user["role"]; // "frontend"
```

언제 대괄호를 쓰는가?
- 키가 변수일 때
- 공백/특수문자가 포함된 키

```js
const key = "name";
user[key];
```

---

## 3. 객체 추가 / 수정 / 삭제

```js
user.age = 29;          // 추가
user.role = "senior";  // 수정
delete user.active;    // 삭제
```

⚠️ 실무 주의
- React/상태 관리에서는 **직접 수정 지양**
- 불변성 유지 필요

---

## 4. 객체와 참조(reference)

```js
const a = { x: 1 };
const b = a;

b.x = 2;
console.log(a.x); // 2
```

객체는 **값 복사 ❌ / 참조 공유 ⭕**

---

## 5. 객체 복사 (얕은 복사)

```js
const copy = { ...user };
```

또는

```js
const copy = Object.assign({}, user);
```

⚠️ 중첩 객체는 여전히 참조 공유 (얕은 복사)

---

## 6. 구조 분해(Destructuring)란?

구조 분해 = **객체에서 필요한 값만 빠르게 꺼내는 문법**

```js
const { name, role } = user;
```

위 코드는 아래와 동일:

```js
const name = user.name;
const role = user.role;
```

---

## 7. 변수명 변경 (alias)

```js
const { name: userName } = user;
```

실무에서 매우 자주 사용

---

## 8. 기본값 설정

```js
const { age = 0 } = user;
```

- 값이 `undefined`일 때만 기본값 적용

---

## 9. 중첩 객체 구조 분해

```js
const data = {
  user: {
    profile: {
      nickname: "JJ",
    },
  },
};

const {
  user: {
    profile: { nickname },
  },
} = data;
```

⚠️ 가독성 저하 시 분리 권장

---

## 10. 함수 매개변수 구조 분해

```js
function renderUser({ name, role }) {
  return `${name} (${role})`;
}
```

장점:
- 함수 시그니처만 봐도 **필요 데이터 파악 가능**
- 순서 의존 제거

---

## 11. 객체 + spread 패턴 (실무 핵심)

```js
const nextUser = {
  ...user,
  role: "lead",
};
```

- 기존 객체 유지
- 변경 지점만 덮어쓰기

React, 상태 관리의 기본 패턴

---

## 12. Rest 패턴 (나머지 모으기)

```js
const { id, ...rest } = user;
```

- 특정 키 제외한 나머지 객체 생성

---

## 13. 언제 객체 & 구조 분해를 쓰는가 (정리)

- API 응답 처리
- 컴포넌트 props
- 상태 업데이트
- 옵션 객체 전달
- 설정(config) 관리

👉 **JS 실무의 절반은 객체 구조 분해**

---

## 14. 실무 사고 요약

- 객체 = 의미 단위 데이터
- 구조 분해 = 데이터 접근 비용 제거
- spread/rest = 불변성 유지 도구
- 읽기 쉬운 구조가 좋은 코드
