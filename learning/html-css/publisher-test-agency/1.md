# Day 14 — Starbucks Korea Renewal: Winter Full-page Responsive Multi-page

**Date:** 2026-01-08  
**Topic:** HTML / CSS  
**Goal:** 스타벅스 코리아 웹사이트를 레퍼런스로 삼아, **겨울 시즌 무드의 완전 리뉴얼 멀티 페이지**를 설계/구현한다. (퍼블리셔 전형 기준: 구조·레이아웃·반응형·상태·모션·아키텍처)

---

## 📌 프로젝트 개요

이 프로젝트는 Day 1 ~ Day 13까지 학습한 HTML / CSS 핵심 개념을 종합 적용하여  
**스타벅스(https://www.starbucks.co.kr/index.do) 기반의 “완전 리뉴얼” 멀티 페이지 사이트**를 제작하는 퍼블리셔 실전 과제다.

단순 클론이 아니라,

- 브랜드/콘텐츠 구조를 분석한 뒤
- **완전히 새로운 레이아웃 시스템**으로 재구성하고
- **겨울 시즌 컬러 토큰**을 중심으로 UI 일관성을 만들며
- 페이지 수가 늘어나도 유지되는 CSS 아키텍처를 구축한다.

핵심은 결과물이 아니라 **설계 과정**이다.

## 🎯 미션 목표

- **7일 스프린트**로 멀티 페이지 “풀페이지” 구현 (PC / Tablet / Mobile)
- 공통 레이아웃(헤더/푸터) + 페이지별 콘텐츠 확장 구조 설계
- Grid + Flex 기반 레이아웃 시스템 구축
- Selector / Variable 기반 상태 처리(hover / focus-visible / active / disabled)
- Transition / Easing 기반 미세 모션 적용(실무형)
- 이미지 레이아웃에서 CLS 방지(비율 박스/사이즈 고정)
- 유지보수 가능한 CSS 아키텍처(토큰 → 베이스 → 레이아웃 → 컴포넌트 → 페이지)

## ✅ 오늘의 핵심

- 리뉴얼은 “예쁘게”가 아니라 **구조를 다시 짜는 작업**이다
- 컬러는 감성이 아니라 **토큰 시스템**이다
- 멀티 페이지는 페이지가 늘어날수록 **규칙의 힘**이 드러난다
- 레이아웃은 Grid, 정렬은 Flex, 상태는 Selector/Variable, 모션은 Transform/Opacity

## ✅ 구현 기준 & 이 과제를 통해 기른 역량

### 내가 구현하면서 지키려던 기준

- 모든 페이지에서 동일한 HTML 골격 유지 (Header / Main / Footer)
- **새 레이아웃 시스템**을 먼저 확정하고 페이지에 적용
- 불필요한 wrapper / 클래스 추가 금지 (역할 단위 최소 구조)
- 컬러/여백/타이포/라운드/쉐도우 전부 토큰화
- 상태 변화는 Selector 우선(필요 시 최소 클래스)
- 반응형은 단순 축소가 아니라 **레이아웃 재배치**
- 접근성 기본: heading hierarchy / focus-visible / aria-label(필요 시)

### 이 과제를 통해 기른 역량

- 브랜드 사이트 리뉴얼 관점의 IA(정보 구조) 설계
- Grid/Flex 기반 “페이지 프레임” 구축 능력
- 디자인 토큰 기반 CSS 시스템 설계
- 상태/모션의 일관성(컴포넌트 레벨) 유지 능력
- 멀티 페이지 유지보수(규칙 중심) 경험

## 🛠 구현 내용

### 1) 겨울 시즌 컬러 시스템(Design Tokens)

- 브랜드 고유의 “그린”은 유지하되, 겨울 무드를 위해 **차가운 중성 톤 + 포인트 컬러**를 추가한다.

권장 팔레트(예시):
- Evergreen(브랜드 메인): `--brand-600`
- Snow(배경): `--bg-0`
- Charcoal(텍스트): `--text-900`
- Cranberry(겨울 포인트): `--accent-600`
- Ice Blue(보조 포인트): `--accent-300`

토큰 예시(초안):
```css
:root {
  --bg-0: #fbfbfc;
  --bg-50: #f2f4f7;

  --text-900: #111318;
  --text-600: #4b5563;

  --brand-600: #0b6b3a;    /* evergreen */
  --brand-700: #085a31;

  --accent-600: #b11f2a;   /* cranberry */
  --accent-300: #8fc7ff;   /* ice blue */

  --radius-md: 14px;
  --shadow-sm: 0 6px 18px rgba(0,0,0,.08);

  --space-2: 8px;
  --space-3: 12px;
  --space-4: 16px;
  --space-6: 24px;
  --space-8: 32px;

  --container: 1200px;
}
```

### 2) 페이지 구조(IA) — 멀티 페이지 “풀페이지”

- `Home`(랜딩): 브랜드/시즌 캠페인 + 핵심 진입 허브
- `Menu`: 음료/푸드/MD 카테고리 그리드
- `Store`: 매장 찾기(레이아웃 중심 페이지), 매장 유형 소개 섹션
- `Rewards`: 리워드 소개/혜택/가입 CTA
- `Brand`: 브랜드 스토리/가치/사회공헌(콘텐츠 페이지)
- `Contact`: 문의 폼(접근성/상태/UI)

### 3) 7일 작업 플랜(권장)

- Day 1: IA 확정 + 와이어(섹션) + 토큰/베이스 세팅
- Day 2: 공통 레이아웃(Header/Footer) + 네비게이션 반응형
- Day 3: Home 페이지(섹션 6~8개) 1차 완성
- Day 4: Menu 페이지(카테고리/카드 시스템) + 공통 컴포넌트 보강
- Day 5: Store/Rewards 페이지 + 상태/모션 정리
- Day 6: Brand/Contact 페이지 + 접근성/CLS 체크
- Day 7: 폴리싱(반응형 디테일/타이포/간격) + README 스냅샷 정리
