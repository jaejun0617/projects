# Day 14 — Responsive Landing Multi-page

**Date:** 2026-01-08  
**Topic:** HTML / CSS  
**Goal:** Day 1 ~ Day 13에서 학습한 레이아웃, 상태, 모션, 구조 설계 개념을 적용하여 퍼블리셔 전형 기준의 반응형 랜딩 멀티 페이지 사이트를 구현한다.

---

## 📌 프로젝트 개요

이 프로젝트는 Day 1 ~ Day 13까지 학습한 HTML / CSS 핵심 개념을 종합 적용한  
퍼블리셔 실전형 반응형 랜딩 멀티 페이지 프로젝트다.

단일 페이지 예제가 아닌,

- 공통 레이아웃을 공유하는 멀티 페이지 구조
- 실제 서비스 형태에 가까운 콘텐츠 구성
- 반응형 · 상태 · 모션 · 유지보수를 고려한 CSS 설계

를 목표로 한다.

핵심은 결과물이 아니라 설계 과정이다.

---

## 🎯 미션 목표

- 반응형 랜딩 페이지 구현 (PC / Tablet / Mobile)
- 멀티 페이지 구조 설계 (공통 Header / Footer)
- Grid + Flex 기반 레이아웃 구성
- Selector / Variable 기반 상태 처리
- CSS Transition / Animation을 활용한 UI 모션 적용
- 구조 변경 없이 확장 가능한 CSS 설계

---

## ✅ 오늘의 핵심

- 레이아웃은 배치가 아니라 계산이다
- CSS는 스타일링이 아니라 구조 설계 도구다
- 값은 직접 쓰지 않고 토큰으로 관리한다
- 상태와 모션은 CSS로 충분히 표현 가능하다

## ✅ 구현 기준 & 이 과제를 통해 기른 역량

### 내가 구현하면서 지키려던 기준

- 모든 페이지에서 동일한 HTML 골격 유지
- 레이아웃은 Grid, 내부 정렬은 Flex로 역할 분리
- 불필요한 wrapper / 클래스 추가 금지
- 색상 · 여백 · 폰트 크기 CSS Variables 토큰화
- 상태 변화는 Selector 기반으로 처리
- 반응형은 단순 축소가 아닌 재배치 중심 설계

### 이 과제를 통해 기른 역량

- 시맨틱 HTML 멀티 페이지 구조 설계
- 레이아웃 엔진(Grid / Flex) 분리 사고
- CSS Selector 기반 상태 제어 능력
- 디자인 토큰 기반 CSS 아키텍처 감각
- 퍼블리셔 실전 기준 문제 해결 경험
