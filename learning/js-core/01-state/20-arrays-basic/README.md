- UI는 배열 상태를 그대로 반복 렌더한 결과물이다
- 이 구조는 모든 리스트 UI의 기본형이다

## 🧠 핵심 개념 / 설계 요약

### 1️⃣ 배열 = 상태 리스트
배열은 여러 개의 동일한 의미를 가진 상태를 순서대로 관리한다.

```js
const state = {
  todos: [],
};
```

UI는 이 배열을 직접 수정하지 않고,
상태 변경 후 항상 render를 다시 호출한다.

---

### 2️⃣ Todo 엔티티 객체
배열의 각 요소는 단순 문자열이 아니라 **의미를 가진 객체**다.

```js
function createTodo(text) {
  return { text };
}
```

이 구조는 이후:
- 완료 여부(isDone)
- 생성 시간(createdAt)

같은 속성을 추가할 수 있는 확장 지점이 된다.

---

### 3️⃣ 반복 렌더링
반복문은 “여러 번 실행”이 아니라,
**리스트 구조를 화면에 생성하는 규칙**이다.

```js
for (let i = 0; i < state.todos.length; i++) {
  // index 기반 UI 생성
}
```
