## 🧭 Day 15–60 전체 흐름 요약

### Phase 1 — State 사고 형성 (Day 15–25)
- 상태 → 렌더 사고 체득
- 조건 / 반복 / CRUD / 필터 / Guard
- Todo UI를 통한 상태 중심 사고 완성

### Phase 2 — Event & Interaction (Day 41–45)
- 이벤트 아키텍처
- Delegation / Accessibility / Cleanup
- 이벤트 = 상태 변경 요청 개념 확립

### Phase 3 — Async & Data (Day 46–49)
- 비동기 흐름 제어
- Error / Race / Cancel
- 데이터 기반 UI 설계

### Phase 4 — SPA & Routing (Day 50–55)
- URL = State
- Router 구현
- Vite 기반 SPA 실전 완성

### Phase 5 — Architecture 통합 (Day 56–60)
- 상태 아키텍처 언어화
- React 필요성 이해
- Hook 사고 연결
- 성능 & 리팩터링 기준 정립
